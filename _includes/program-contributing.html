Contributing to Ghana in Data

This is a summary of how we approach contributing to open source at Ghana in Data.

We welcome and encourage contributions to Ghana in Data! Whether you're a developer or data expert, contributing helps improve the project and keeps it valuable for everyone.

Here's How to get started:

  • Check Out the Repo: Visit our GitHub repository to explore the codebase. Fork the repo, clone it locally, and follow the setup instructions in the README to get started.
  • Good First Issues: If you’re new to contributing, we’ve labeled some beginner-friendly issues with "Good First Issue" to help you ease into the project. Tackle these to get familiar with the codebase while making valuable contributions.
  • Small changes: For minor updates like bug fixes or enhancements, fork the repository make your changes, and submit a pull request. Be sure your commit messages are clear and follow our guidelines.
  • Larger changes: For substantial contributions (new features or major revisions), follow the contributor guidelines and submit a pull request for review. We’ll provide feedback and ensure your changes align with the project’s goals.
  • Engage the Community: Before submitting changes, take time to understand our community’s practices. Be sure to communicate clearly and engage respectfully.
  • Engage the Community: Note that different open source communities take contributions in different ways. Before firing off a pull request, investigate how best to engage the community you will be contributing back to.
  • Code Review & Standards All contributions go through a review process to ensure quality. Adhere to our standards—clean code, meaningful commit messages, and thorough documentation.
  • Fork Nicely: Always try to contribute to a community instead of hard forking. Review internal guidelines and training before forking into an official GitHub org.

Ready to get started? Check out the repo and take your first step!
